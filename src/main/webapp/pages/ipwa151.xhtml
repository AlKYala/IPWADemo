IPWA1: 5-1 Java Beans

Es geht in diesem Kapitel hauptsaechlich darum, die Lebensdauer von den einzelnen Beans darzustellen. 

An dieser Stelle eine kurze Erklärung zu den verhaltensweisen der Beans:

None Scoped: Es wird nichts passieren, da nach der Request die Aenderungen sterben

Request Scoped: Sobald du ein neues Request machst, gehen die alten aenderungen weg, da damit die Lebensdauer des alten Requests endet (anderer Req. startet)

View Scoped: Bleibt so lange wie du den Tab hast. Dementsprechend uebertragen sich die Aenderungen nicht auf andere Tabs

Session Scoped: Aenderungen bleiben Tabuebergreifend,  da selbe Session. Neue Sessions werden z.B. durch einen anderen Browser / Incognito Tab erzeugt.

Application Scoped: Aenderungen sind serverseitig - solange der Server nicht down geht, bleiben die Änderungen.
